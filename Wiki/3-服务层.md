# 3. 服务层设计

## 3.1 模板服务

### ITemplateService（接口）
**文件位置**: `WorkLogApp.Services/Interfaces/ITemplateService.cs:117-`
```csharp
// 接口定义包含：
// - 加载模板
// - 保存模板  
// - 渲染模板
// - 获取分类模板
// - 新增/更新/删除分类模板
```

### TemplateService（实现）
**文件位置**: `WorkLogApp.Services/Implementations/TemplateService.cs`

#### 核心方法
1. **模板加载**（行16）
   - 从`Templates/templates.json`加载模板配置

2. **模板保存**（行26）
   - 将模板配置保存到JSON文件

3. **模板渲染**（行45-81）
   - 支持`{字段}`与`{字段:格式}`占位符
   - 日期格式优先，回退`ToString()`
   - 系统字段注入：`{ItemTitle}`, `{CategoryPath}`等（行75-80）

4. **分类层级合并**（行93-160）
   - 将`父-子-孙`名称链中的模板格式、占位符与选项合并
   - 格式文本以空行分隔拼接

5. **模板管理**（行179, 187）
   - 新增/更新分类模板（行179）
   - 删除分类模板（行187）

## 3.2 导入导出服务

### IImportExportService（接口）
**文件位置**: `WorkLogApp.Services/Interfaces/IImportExportService.cs:7-13`

### ImportExportService（实现）
**文件位置**: `WorkLogApp.Services/Implementations/ImportExportService.cs`

#### 核心功能
1. **导出合并策略**（行40-64, 67-85）
   - 对同月数据合并现有与新增
   - 按天聚合后覆盖写入

2. **Excel写入细节**：
   - 日期块标识行：`===== yyyy年MM月dd日 星期X =====`（行219-230）
   - 事项排序：`SortOrder` → `StartTime` → `ItemTitle`（行231-235）
   - 当日总结行（行265-277）
   - 列宽与样式设置（行280-288）

3. **分类ID写入策略**（行90-107, 246-255, 566-582）
   - 优先写模板名称，失败回退为数值ID
   - 名称到稳定ID映射通过FNV-1a哈希算法

4. **数据导入**：
   - 从目录按月读取（行305-399）
   - 从文件读取（行401-491）
   - 识别日期块与"当日总结"行（行324-340, 343-356）
