# 工作日志桌面应用绿色化实施方案

## 目标
将现有 WinForms 应用（WorkingLog）改造为“绿色软件”，满足：
1. **单文件运行**：只有一个可执行文件（WorkLogApp.UI.exe），无外部 DLL、无配置文件。
2. **首次启动自动释放**：将嵌入的配置文件（dev.config.json、prod.config.json、templates.json）释放到可执行文件所在目录的对应子目录中。
3. **数据目录自动创建**：在可执行文件同目录下创建 Data 文件夹（若不存在），用于存放 Excel 日志文件。

## 技术方案
### 1. 嵌入资源配置
- 将以下文件添加为嵌入式资源（EmbeddedResource）：
  - `Configs/dev.config.json`
  - `Configs/prod.config.json`
  - `Templates/templates.json`
- 保留原有的 Content 条目，确保开发时仍能复制到输出目录。
- 修改 `WorkLogApp.UI.csproj`，添加对应的 `<EmbeddedResource>` 项。

### 2. 资源管理器类
创建 `ResourceManager` 静态类，提供以下方法：
- `EnsureFileFromResource(string resourcePath, string outputPath)`：检查输出路径是否存在，若不存在则从程序集资源中提取并写入。
- `EnsureDirectory(string path)`：确保目录存在。

### 3. 启动时初始化
在 `Program.Main()` 中添加初始化步骤：
- 确定基础目录 `AppDomain.CurrentDomain.BaseDirectory`。
- 调用 `ResourceManager.EnsureFileFromResource` 依次释放三个配置文件到 `Configs/` 和 `Templates/` 子目录。
- 创建 `Data` 文件夹（若不存在）。

### 4. 配置文件读取逻辑
保持现有的配置读取方式不变（通过 `ConfigurationManager.AppSettings` 和 JSON 文件），因为释放后的文件位于相同路径，代码无需修改。

### 5. 单文件构建
- 依赖已有的 Costura.Fody 将所有引用的 DLL 嵌入主程序集。
- 发布时使用 Release 配置，确保输出目录只包含 exe 文件（无其他文件）。
- 验证单独复制 exe 到空白目录后能正常启动并自动生成所需文件和文件夹。

## 实施步骤
1. 修改项目文件（添加 EmbeddedResource）
2. 创建 ResourceManager 类
3. 修改 Program.cs 添加初始化调用
4. 测试 Debug 版本是否仍能正常工作
5. 构建 Release 版本，复制 exe 到独立目录进行全功能测试
6. 验证导入/导出、模板加载、数据存储等核心功能

## 注意事项
- 释放文件时需保持原有目录结构（Configs/、Templates/）。
- 若用户已存在同名的配置文件，不应覆盖（保留用户修改）。
- 异常处理：资源提取失败时应有友好提示，但不应阻止应用启动。
- 考虑在日志中记录首次释放的操作，便于排查问题。
